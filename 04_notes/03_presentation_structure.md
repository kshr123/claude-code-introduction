### 2. Claude Codeとは何か（15分）

#### 2.1 概要

**スライド3: Claude Codeとは**
```
Claude Code = ターミナルで動作するAIコーディングアシスタント

✓ 自然言語で指示を出すだけでコードを書いてくれる
✓ ファイル編集、コマンド実行、Git操作まで自動化
✓ プロジェクト全体を理解して適切な提案
✓ テスト・ドキュメントも自動生成
```

**スライド4: 他のツールとの違い**
```
| 機能 | GitHub Copilot | Cursor | Claude Code |
|------|---------------|--------|-------------|
| 実行環境 | エディタ内 | エディタ内 | ターミナル |
| ファイル編集 | 補完 | 全体編集 | 全体編集 |
| コマンド実行 | ✗ | △ | ✓ |
| Git操作 | ✗ | △ | ✓ |
| 複数ファイル | △ | ✓ | ✓ |
| 外部連携(MCP) | ✗ | ✗ | ✓ |
```

**話すこと**:
- ターミナルベースの利点
- 既存ワークフローへの統合
- エディタに依存しない柔軟性

#### 2.2 主要機能

**スライド5: 4つの中核機能**
```
1. 機能開発
   自然言語で指示 → 計画・実装・検証まで自動化

2. デバッグ支援
   エラーメッセージから原因特定 → 修正案提示 → 実装

3. コードベース探索
   プロジェクト構造を理解 → アーキテクチャ説明

4. 自動化
   リント修正、マージ競合解決、リリースノート作成
```

**スライド6: 2025年の最新機能**
```
🔥 サブエージェント（2025年7月）
   専門的なAIチームを編成（レビュアー、テスター、セキュリティ監査）

🔥 スキル機能（2025年10月）
   カスタムワークフローをスキル化してチーム共有

🔥 MCP統合（2025年6月強化）
   Google Drive、Slack、GitHub、Postgres等と連携
```

**話すこと**:
- 継続的に進化している
- 最新のAI技術を活用
- エンタープライズ向け機能の充実

#### 2.3 プロジェクトメモリ（`.claude/CLAUDE.md`）⭐ **重要**

**スライド7: プロジェクトメモリとは**
```
.claude/CLAUDE.md = Claude Codeへの指示書

✓ プロジェクト固有のルールを記載
✓ Claude Codeが自動的に読み込んで従う
✓ チーム全体で共有できる
✓ オンボーディング時間を大幅短縮
```

**スライド8: 従来との違い**
```
【従来のAIツール】
ユーザー: "テストを書いて"
AI: "どんなテストですか？"
ユーザー: "pytestで、カバレッジ80%以上で..."
（毎回説明が必要）

【Claude Code with .claude/CLAUDE.md】
ユーザー: "テストを書いて"
Claude Code: [.claude/CLAUDE.mdを読む]
            → すぐに適切なテストを生成
```

**スライド9: 何を書くべきか**
```
.claude/CLAUDE.mdに記載する内容

1. ディレクトリ構造とルール
2. コーディング規約（言語、スタイル、テスト）
3. 命名規則
4. ワークフロー（開発プロセス）
5. Claude Codeへの指示
```

**スライド10: 実例**
```markdown
# .claude/CLAUDE.md

## コーディング規約
- 言語: Python 3.13
- スタイル: PEP 8
- テスト: pytest（カバレッジ80%以上）
- コメント: 日本語で記載

## Claude Codeへの指示
- Git操作前に必ず確認を取る
- テスト駆動開発を実践する
- ドキュメントは必ず更新する
```

**スライド11: メモリの階層構造**
```
エンタープライズレベル（組織全体）
  ↓
ユーザーレベル（個人設定）
  ↓
プロジェクトレベル（.claude/CLAUDE.md）← 最優先
```

**スライド12: 導入メリット**
```
オンボーディング時間: 50-60%短縮
  → 新人がすぐに戦力に

ルール説明の時間: 80%削減
  → ドキュメントが自動的に効く

コード品質の均一化
  → チーム全体で統一されたコード
```

**話すこと**:
- プロジェクトメモリはClaude Codeの核心機能
- 一度設定すればチーム全体で活用できる
- ナレッジの蓄積と共有が自動化される
- 実際のデモで効果を体感してもらう

### 3. 実演デモ（15分）

#### デモ1: 新機能開発（5分）

**シナリオ**: ユーザー認証機能の追加

**実演内容**:
```bash

claude "JWT認証を追加したい。どう実装すべき?"

# 3. テスト追加
claude "認証機能のテストを追加して"

# 4. ドキュメント作成
claude "README.mdに認証機能の使い方を追加して"

# 5. コミット
claude "変更をコミットして。わかりやすいメッセージで"
```

**ポイント**:
- 自然言語で指示するだけ
- 複数ファイルの編集を自動化
- テスト・ドキュメントも自動生成
- Git操作まで完結

### 6. Q&A（5分）

**想定質問と回答**

**Q1: コストはどのくらいかかりますか？**
```
A: Claude Proプランで約2万円/月/人です。
   エンタープライズプラン（Bedrock/Vertex AI）も選択可能です。
   ROI試算では年間2,200万円の純利益が見込めます。
```

**Q2: セキュリティは大丈夫ですか？**
```
A:
- コードはAnthropicのサーバーで処理されますが、学習には使用されません
- エンタープライズプラン（Bedrock/Vertex AI）なら社内インフラで完結
- APIキーや機密情報の管理ガイドラインを策定します
```

**Q3: 既存のツール（GitHubCopilot等）と併用できますか？**
```
A:
- はい、併用可能です
- Claude Codeはターミナルベース、Copilotはエディタ内補完
- 用途に応じて使い分けることで相乗効果が期待できます
```

**Q4: 学習コストはどのくらいですか？**
```
A:
- 基本操作: 1-2日で習得可能
- 高度な機能（サブエージェント、MCP）: 1-2週間
- チーム全体の標準化: 1-2ヶ月
```

**Q5: どのプログラミング言語に対応していますか？**
```
A:
- 主要な言語はすべて対応（Python, JavaScript, TypeScript, Go, Rust等）
- フレームワークも広範囲にサポート（React, Vue, Django, Rails等）
```

**Q6: .claude/CLAUDE.mdはどのくらいの粒度で書くべきですか？**
```
A:
- 原則とガイドラインを記載（詳細すぎると読みにくい）
- チーム全体で共有したいルールを優先
- 段階的に充実させていくのがおすすめ
- 実例: コーディング規約、命名規則、ワークフロー
```

**Q7: 既存のプロジェクトにも導入できますか？**
```
A:
- はい、既存プロジェクトでも導入可能です
- まず基本的な.claude/CLAUDE.mdを作成
- プロジェクトの既存ルールを文書化
- 段階的に充実させていく
```
