# Claude Codeの紹介 - Claude Rules

## 📋 概要

このリポジトリはClaude Codeを初めて使う人のための実践的なガイドです。
調査、理解、実践を通じて、Claude Codeの使い方と活用方法を体系的にまとめています。

---

## 🎯 目的

1. Claude Codeの基本機能と使い方を理解する
2. 実際のユースケースを収集・整理する
3. 初心者がつまずきやすいポイントを明確にする
4. わかりやすいガイド資料を作成する
5. プロジェクトメモリ（`.claude/CLAUDE.md`）の活用方法を伝える

---

## 📁 ディレクトリ構造とルール

### ディレクトリの役割

```
.
├── 01_reference/           # 公式ドキュメント、参考資料（読み取り専用）
├── 02_templates/           # テンプレートファイル
├── 03_my_implementations/  # （予備：必要に応じて使用）
├── 04_notes/               # 調査ノート・ガイド記事（書き込み可）
├── 05_progress/            # 進捗管理（書き込み可）
└── 06_docs/                # 最終的なガイド資料（書き込み可）
```

### 重要なルール

1. **`01_reference/` は読み取り専用**
   - 公式ドキュメントや参考資料は変更しない
   - 調査や理解のためにのみ参照する

2. **`04_notes/` に調査内容とガイド記事を作成**
   - Claude Codeの機能調査
   - ユースケースの整理
   - 初心者向けガイドの下書き
   - **命名規則**: `{NN}_{topic_name}.md`（連番接頭辞を必ず付ける）
   - **連番の目的**: 学習の流れを明確にする

3. **`05_progress/` で進捗を管理**
   - `learning_log.md` は**進捗の把握のみに使用**
     - どこまで完了したか（チェックリスト）
     - 次にどこから始めるか（次のタスク）
   - **詳細な内容は各ディレクトリのREADMEで確認する**
     - 調査内容、ガイド記事の詳細は各ノートに記載
     - learning_log.mdには詳細を書かず、完了日とステータスのみ記録

4. **`06_docs/` に最終ガイド資料を配置**
   - 初心者向けガイド（統合版）
   - クイックスタートガイド
   - FAQとトラブルシューティング

## 💻 コーディング規約

### Python（デモコード用）

- **バージョン**: Python 3.13以上
- **パッケージマネージャー**: uv
- **スタイル**: PEP 8に準拠
- **型ヒント**: 可能な限り使用する
- **docstring**: 重要な関数・クラスには必須
- **コメントとdocstringの言語**: **必ず日本語で記載すること**
  - ✅ モジュールdocstring: 日本語
  - ✅ 関数/クラスdocstring: 日本語
  - ✅ インラインコメント: 日本語
  - ❌ 英語のコメントは使用しない
- **命名規則**:
  - 関数/変数: `snake_case`
  - クラス: `PascalCase`
  - 定数: `UPPER_CASE`

### Markdown（資料作成用）

- **見出し**: 階層を明確にする
- **コードブロック**: 言語指定を必ず行う
- **リンク**: 相対パスを使用
- **画像**: `06_docs/images/` に配置

## 📝 ガイド作成プロセス

### 作成フロー（全5フェーズ）

#### 1. **調査フェーズ** 📖
   - Claude Code公式ドキュメントを読む
   - 既存のブログ記事やコミュニティ情報を調査
   - ユースケースや事例を収集
   - `04_notes/` に調査ノートを作成

#### 2. **理解フェーズ** 💡
   - 調査した内容を整理・構造化
   - 初心者がつまずきやすいポイントを特定
   - 重要な概念を明確化
   - `04_notes/` に理解を深めたノートを追加

#### 3. **ガイド作成フェーズ** ✍️
   - わかりやすい説明文を作成
   - 具体例とコード例を追加
   - 図表やスクリーンショットを準備
   - `06_docs/` にガイド資料を配置

#### 4. **レビュー・改善フェーズ** 🔍
   - ガイドの内容を確認
   - 初心者の視点でチェック
   - わかりにくい部分を改善
   - FAQ・トラブルシューティングを追加

#### 5. **振り返りフェーズ** 📊
   - **ドキュメント更新（必須）**
     1. 各ノートの内容を確認
     2. プロジェクトルートの `README.md` を更新（全体の進捗を反映）
     3. `progress/learning_log.md` に完了日とステータスを記録
   - **成果物の整理**
     - 最終ガイドを `06_docs/` に配置
     - 不要なファイルを削除
   - **Git操作**
     - 変更をコミット
     - プッシュ（Claude Codeは実行前に必ず確認を取る）

## 📊 進捗管理

### progress/learning_log.md の役割

**重要**: `progress/learning_log.md` は**進捗把握専用**です。詳細な内容は書きません。

**記載内容**:
- ✅ どこまで完了したか（チェックリスト）
- ✅ 次にどこから始めるか（次のタスク）
- ✅ 完了日のみ（例: `2025-11-05`）
- ❌ 資料の詳細内容（各READMEで確認）
- ❌ 調査内容の詳細（04_notes/で確認）

### ドキュメント更新フロー（必須）

**各タスク完了後は必ず以下の順序でドキュメントを更新すること**:

1. **タスクのREADME.md**
   - タスクの詳細、成果物、学んだことを記載
   - 他の人が見てもわかるように丁寧に書く

2. **プロジェクトルートのREADME.md**
   - 全体の完了タスク数を更新
   - 完了したタスクのリストを更新

3. **progress/learning_log.md**
   - 完了日とステータスのみ記録
   - 詳細は書かない

## 🤝 Claude Codeとの協働ルール

### Claude Code に期待すること

1. **ガイド構成支援**: わかりやすいガイド構成の提案
2. **コンテンツ作成**: 初心者向けの説明文の作成補助
3. **例示**: 具体的な使用例とコマンド例の提供
4. **レビュー**: ガイドの改善提案
5. **FAQ作成**: よくある質問と回答の作成補助

### Claude Code が守るべきこと

1. **段階的作成**: 一度に全てを作成せず、小さなステップで進める
2. **わかりやすさ重視**: 専門用語を避け、平易な表現を使う
3. **具体性**: 抽象的な説明ではなく、具体例を示す
4. **実践的**: すぐに試せる情報を提供する
5. **Git操作の確認**: `git push` は実行前に必ずユーザーの確認を取る
6. **ドキュメント更新**: 各タスク完了後、**必ず3段階のドキュメント更新**を実施する
   - ノートの内容確認 → README.md更新 → learning_log.md更新
7. **命名規則**:
   - ノートファイルには連番を接頭辞として付ける（01_, 02_, 03_...）

## 🔒 個人情報とセキュリティルール

### 個人情報の取り扱い

このリポジトリは公開リポジトリです。個人情報の管理を徹底してください。

#### 禁止事項

以下をコード、ドキュメント、コミットメッセージに含めないでください：

1. **実名**
2. **連絡先情報**（メールアドレス、電話番号）
3. **所属情報**（会社名、部署名）

#### 推奨事項

**匿名化を徹底**:
- 例示には架空の名前を使用
- スクリーンショットには個人情報を含めない
- 一般的な例を使用

### 機密情報の取り扱い

#### 管理が必要な情報

1. **APIキー・トークン**
2. **環境変数**
3. **秘密鍵**

#### 安全な管理方法

**環境変数ファイル**:
```bash
# .env ファイルを使用（gitignore必須）
API_KEY=your_key_here
```

**サンプルファイルの提供**:
```bash
# .env.example をリポジトリにコミット
API_KEY=your_key_here
```

**.gitignoreの設定**:
```gitignore
# Secrets
*.pem
*.key
credentials.json

# MCP Configuration
.mcp.json

# Environment variables
.env
.env.local
```
